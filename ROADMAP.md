# ROADMAP.md

## MVP 定义

MVP 完成标准：

- Phaser + TypeScript + Vite 客户端可运行。
- Colyseus + Node.js + TypeScript 服务端可运行。
- 2 名远程玩家可进入同一房间。
- 服务端权威处理输入、碰撞、机关和通关。
- 10 个关卡可从头到尾完成。
- 项目可部署到自有 VPS。
- 基础测试、构建和部署文档齐全。

## 阶段 0：文档与约束

目标：锁定开发边界，避免后续实现跑偏。

任务：

- R0-01：创建 `AGENTS.md`，定义 Codex 开发规则。
- R0-02：创建 `GAME_DESIGN.md`，定义 MVP 玩法和关卡方向。
- R0-03：创建 `NETWORK_SPEC.md`，定义服务器权威协议。
- R0-04：创建 `LEVEL_SCHEMA.md`，定义关卡 JSON schema。
- R0-05：创建 `ROADMAP.md`，拆分可验收任务。

验收：

- 5 个文档存在。
- 文档之间术语一致。
- 没有实际游戏代码。

## 阶段 1：项目脚手架

目标：建立可运行、可测试、可部署的基础工程。

任务：

- R1-01：创建 npm workspace 或等价 monorepo 结构。
- R1-02：创建 `apps/client/` Vite + Phaser + TypeScript 项目。
- R1-03：创建 `apps/server/` Colyseus + TypeScript 项目。
- R1-04：创建 `packages/shared/` 共享类型包。
- R1-05：配置 `typecheck`、`lint`、`test`、`build`。
- R1-06：配置开发环境变量示例 `.env.example`。

验收：

- `npm install` 成功。
- `npm run dev:client` 能打开空 Phaser 场景。
- `npm run dev:server` 能启动 Colyseus 服务。
- `npm run typecheck`、`npm run build` 成功。

## 阶段 2：联机房间闭环

目标：完成 2 人房间从创建到 ready 的最小闭环。

任务：

- R2-01：实现创建房间并生成房间码。
- R2-02：实现通过房间码加入。
- R2-03：限制每个房间最多 2 人。
- R2-04：实现 playerIndex 分配。
- R2-05：实现 ready 状态。
- R2-06：实现房间状态 UI。

验收：

- 两个浏览器可进入同一房间。
- 第三个浏览器加入时被拒绝。
- 两名玩家 ready 后房间进入加载关卡阶段。

## 阶段 3：服务器权威移动

目标：客户端发送输入，服务端广播玩家状态。

任务：

- R3-01：定义 shared 输入协议类型。
- R3-02：客户端采集方向、跳跃、交互输入。
- R3-03：客户端按 `NETWORK_SPEC.md` 发送输入序号。
- R3-04：服务端校验输入频率和序号。
- R3-05：服务端模拟玩家移动和重力。
- R3-06：客户端渲染本地和远程玩家。
- R3-07：客户端实现服务端位置校正。

验收：

- 两个浏览器都能看到两名玩家移动。
- 客户端不能通过伪造位置移动角色。
- 断续输入不会导致服务端崩溃。

## 阶段 4：关卡加载与基础碰撞

目标：用 JSON 数据驱动关卡。

任务：

- R4-01：实现 `LEVEL_SCHEMA.md` 的 schema 校验。
- R4-02：实现关卡文件加载。
- R4-03：实现 solid 平台碰撞。
- R4-04：实现 oneWay 平台碰撞。
- R4-05：实现两个玩家出生点。
- R4-06：实现出口区域检测。

验收：

- 服务端能加载 `level-01.json`。
- 非法关卡会给出明确错误。
- 两名玩家可站在平台上并到达出口。

## 阶段 5：机关系统

目标：实现 MVP 所需机关实体。

任务：

- R5-01：实现压力按钮。
- R5-02：实现交互按钮。
- R5-03：实现定时按钮。

- R7-02：实现重开当前关卡投票。
- R7-03：实现延迟/连接状态显示。
- R7-04：优化远程玩家插值。
- R7-05：完善错误提示。
- R7-06：补齐关键自动化测试。

- 一名玩家刷新页面后可在 30 秒内回到原房间。
- 网络抖动时玩家位置不会频繁大幅跳变。
- 房间错误能在客户端明确展示。

## 优先级

必须先做：

- 项目脚手架。
- 2 人房间。
- 服务器权威输入。
- 关卡 schema 校验。
- 基础平台、按钮、门、出口。

可以延后：

- 美术资源。
- 音效。
- 动画细节。
- 复杂 UI。
- 排行榜、账号、匹配。

明确不做：

- 3 人及以上联机。
- P2P 网络。
- 客户端权威判定。
- 移动端专项适配。
- 关卡编辑器。
